05

Mode classification

FunctionalSSE · intake.classified

trust auto-classifies whether this job builds software or runs a process over a corpus. Streamed on /jobs/{id}/events; the recommendation is confirmable below.

Build recommended

Produces a versioned automation (modules + interfaces + DAG). The screening + scoring logic is net-new code targeting the runtime contract.

Process alt

Fan-out over a fixed corpus with per-unit steps + aggregation. Fits the ~200-application backlog, but the reusable artifact is what's asked for.

i
Rationale (streamed): "Customer wants a durable, re-runnable onboarding automation, not a one-shot pass over the backlog — recommend build; the backlog becomes the first operate run." Persists to spec.mode.recommended; confirmed set at the gate below.
