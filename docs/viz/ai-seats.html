The AI Layer
eight seats, one boundary

Nxcleus builds and runs automated business processes without a customer's confidential data ever leaving their walls. The intelligence is organized as eight seats — named roles, each with a data-class clearance and a routing policy. Application code addresses seats, never models: swapping a model is a YAML edit, not a refactor.

Every seat resolves to a backend in one of four zones — LOCAL self-hosted MI300X, AMD_HOSTED Fireworks fallback, EXTERNAL the Anthropic API, and CUSTOM customer BYOK endpoints — and carries a hard data-class ceiling of RAW or SANITIZED. The whole product claim rests on a single fact enforced in code: the only designed-in non-local dispatch is the planner, and all it ever sees is a sanitized brief. Everything else runs on the customer's own metal.

You classify a described business process into one of three execution modes for the Nxcleus engine. Decide from the spec alone.

SYSTEM_BRIEFthe load-bearing translation — strip values, keep structure▶

It sees only what you write here. This is the single most consequential translation in the system: your job is maximum specification fidelity per token, under the RedactionPolicy.

The governing principle: STRIP VALUES, KEEP STRUCTURE. The planner does not need a real customer name, account number, table name, or file path to design good work — it needs to know the SHAPE of the problem exactly. So:

The registry is extensible at runtime. A customer adds an API key, points it at an OpenAI-compatible endpoint, registers models under it with capability flags, and rebinds any seat — Fable 5 is the default planner, not a hardcode. Capability routing treats custom models uniformly.

_resolve_override(seat, scope) looks up a seat_overrides row (per-job, then global), joins to custom_models + api_connections, and returns a Binding(backend="custom", …) plus the connection's zone, ceiling, counts_as_local, and encrypted key ref. A connection's data_class_ceiling defaults to SANITIZED — a third-party API is outside the boundary.
